## 1. 152-ФЗ (персональные данные) — 2025-2026

### 1.1 Согласие отдельным документом (с 1.09.2025)

ФЗ-233 (от 8.08.2024, поправки в 152-ФЗ ст. 9): форма согласия должна быть **отдельной от пользовательского соглашения / оферты**.

**Минимальные элементы (ст. 9 ч. 4)**:
1. ФИО / адрес / реквизиты документа субъекта.
2. ФИО / адрес оператора (наш SaaS И отель — оба).
3. Цель обработки.
4. **Перечень категорий ПД** (ФИО, паспорт, телефон, email, фото, IP, cookies-идентификатор).
5. Перечень действий (сбор, запись, систематизация, накопление, хранение, уточнение, извлечение, использование, передача, обезличивание, блокирование, удаление, уничтожение).
6. Срок действия согласия + способ отзыва.
7. Информация о трансграничной передаче.
8. Информация о третьих лицах (channel managers, ЕПГУ, YooKassa).

### 1.2 РКН-регистрация оператора ПД

- **Обязательна для нас как SaaS-вендора** — да. Мы оператор по поручению (ст. 6 ч. 3) **И** оператор-обработчик ПД сотрудников + ПД клиентов.
- **Каждый отель-tenant** обязан регистрироваться отдельно как оператор ПД своих гостей.
- Уведомление подаётся **до начала обработки** (форма на pd.rkn.gov.ru, бесплатно, 30 дней рассмотрения).
- Изменения целей / категорий / трансгранички — подаём в течение **10 рабочих дней**.

### 1.3 Локализация ПД РФ (ст. 18 ч. 5)

- Первичная запись + накопление + хранение + уточнение + извлечение ПД граждан РФ — **только в БД на территории РФ**.
- Yandex Cloud (`ru-central1-a/b/d`) — **полностью покрывает**.
- AWS / GCP / Azure для ПД граждан РФ — **запрещено**.
- В нашем стеке: YDB + S3 в YC = OK; **Sentry SaaS, Datadog, LogRocket — нельзя**, только Yandex Monitoring + Monium.

### 1.4 DPA (Data Processing Agreement) — наша обязательность

С каждым внешним получателем нужно:
- DPA — поручение на обработку (ст. 6 ч. 3).
- Имя получателя в согласии гостя.
- Проверка наличия у получателя статуса оператора ПД в реестре РКН.

Получатели в нашем кейсе:
- YooKassa (Сбер) — оператор ПД ✓.
- TravelLine / Bnovo / Островок / Я.Путешествия — операторы ПД ✓.
- ЕПГУ (через МВД) — государственный получатель, регламентирован 109-ФЗ.
- Yandex Vision (OCR) — оператор ПД, **нужен DPA с YC**.
- Postbox (Yandex Cloud Postbox) — оператор ПД, DPA в составе YC-договора.

### 1.5 Биометрические ПД (ст. 11)

- Фото лица для face-recognition / face-match / liveness — биометрия → **отдельное письменное согласие**.
- ЕБС (Единая биометрическая система) — обязательное хранение биометрии для определённых сценариев (банки, госуслуги). Для отельной face-match — пока не обязательно.
- Фото в паспорте при OCR — это копия документа, **не биометрия в чистом виде** (если не делаем шаблон лица).

### 1.6 Штрафы 2026 (КоАП ст. 13.11) — после 420-ФЗ от 30.11.2024

| Нарушение | ИП | Юрлицо |
|---|---|---|
| Несвоевременное уведомление РКН | 5–10к | 30–50к |
| Обработка без согласия | **100–300к** | **300–700к** |
| Утечка 1–10к субъектов | 200–400к | **3–5 млн** |
| Утечка 10к–100к субъектов | 300–500к | **5–10 млн** |
| Утечка 100к+ субъектов | 1–1.5 млн | **10–15 млн** |
| **Повторная утечка (оборотный штраф)** | — | **0.1–3% оборота, max 500 млн** |
| Несообщение об утечке в 24ч | 100–400к | 1–3 млн |

### 1.7 Reporting инцидента (data breach)

- В РКН + ГосСОПКА (ФСБ): **уведомление в течение 24 часов** + **расследование в течение 72 часов**.
- Это требование 152-ФЗ ч. 3.1 ст. 21 (введено 30.11.2024 ФЗ-420).
- Форма: pd.rkn.gov.ru → раздел «Сообщение об инциденте».
- ГосСОПКА — через НКЦКИ (https://safe-surf.ru) или ИБ-договор с провайдером.

**В коде**: incident-response runbook + alert-канал в Monium → on-call инженер → юрист → 24h timer.

### 1.8 Транс-граничная передача (ст. 12)

- В страны с «адекватной защитой» — уведомление РКН **ДО начала** + согласие субъекта.
- В страны без — отдельное письменное согласие + те же 24h breach.
- Booking от иностранцев: гость из Турции → ПД в нашу YDB РФ → это **обратная локализация** (НЕ трансграничка).

### 1.9 Реализация в коде

```
- onboarding wizard: step "DPA signing" → blocking gate
- consent storage: separate `consent_documents` table, append-only, with hash/version
- consent renewal: track expiry, prompt re-consent
- right to be forgotten: `pii_purge` Worker, cascade delete + log retention
- breach alerting: Monium → PagerDuty-like → 24h SLA timer
- pd.rkn registration: документ в admin-панели tenant'а с status + ссылка
```

## 2. 54-ФЗ + ОФД + ФФД 1.2

### 2.1 Обязательные tags (FFD 1.2)

| Тэг | Назначение | Обязательно |
|---|---|---|
| 1054 | Признак расчёта | да |
| 1055 | Применяемая СНО | да (если несколько на ИП) |
| 1059 | Наименование предмета расчёта | да |
| 1199 | Ставка НДС | да |
| 1212 | Признак предмета расчёта (4 — услуга) | да |
| 1214 | Признак способа расчёта | да |
| 1008 | Email/телефон покупателя | да (если бумажный не выдаётся) |
| 2108 | Доп. реквизит чека (для тур.налога) | опционально |

### 2.2 Multi-tenant: чей чек

**Чек пробивается от имени отеля** (ИП/юрлицо), НЕ от нашего SaaS. Реквизиты: ИНН отеля, наименование, адрес расчёта (адрес отеля), регистрационный номер ККТ отеля.

В коде: каждый tenant вводит свои ККТ-реквизиты + auto-import из ЮKassa Чеки.

### 2.3 НДС 22% с 1.01.2026

- Базовая ставка ↑ **20% → 22%** с 1.01.2026 (376-ФЗ).
- Новые коды (тэг 1199):
  - `vat_code 11`: НДС 22%
  - `vat_code 12`: расчётная 22/122
  - `vat_code 5`: гостиничные услуги — **0% до 31.12.2030** (продлено).
- F&B и прочее — 22%.
- УСН без НДС → `vat_code 6`.

### 2.4 Возврат и коррекция

- **Возврат прихода (refund)**: тэг 1054 = 2. Ссылка на оригинальный чек.
- **Чек коррекции** (ст. 4.3 п. 4): обязателен при ошибке задним числом.
- VAT rate на refund receipt = VAT rate на original (locked, даже после rate change).

### 2.5 Признак агента

При booking через Booking/Островок/Я.Путешествия: гость платит OTA, OTA удерживает commission, остальное переводит отелю.
- Чек пробивает **отель** (не OTA, не SaaS).
- Тэг 1057 = `0` (отель — принципал, не агент).
- НО если отель сам агент (продаёт ж/д билет) — тэг 1057 = 1–7.

### 2.6 ЮKassa Чеки vs внешние

| Решение | Pros | Cons |
|---|---|---|
| **ЮKassa Чеки** (наш default) | Облачная касса, FFD 1.2 OOTB, fiscalization вместе с платежом, ru-central1 | Привязка к YooKassa; commission; ограничения по объёму |
| **Атол.Онлайн** | Независимость от PSP; больший объём | Дополнительная подписка; интеграция отдельно |
| **OFD.ru ОФД** | Дешевле | Только ОФД, нужна ККТ железка |

**Default: ЮKassa Чеки** для V1. Architectural seam в `PaymentProvider.capabilities.fiscalization: 'native'|'external'|'none'` уже зарезервирован.

### 2.7 Самозанятые + ПСН

- **Самозанятые (НПД)** — могут оказывать услуги краткосрочной аренды (ст. 4 ч. 2 422-ФЗ) — гостевые дома/квартиры. **ККТ НЕ нужна** — выдача чека через **Мой налог** (mobile app + API). Оборот max 2.4 млн ₽/год. Без работников.
- **ПСН (патент)** — гостиничные услуги в перечне, до 60 млн ₽/год, до 15 сотрудников. ККТ ОБЯЗАТЕЛЬНА.
- **Гостевые дома** часто на НПД, **отели** — на УСН/ОСН/ПСН.

В коде: enum `tax_regime` ∈ `{NPD, PSN, USN_DOHODY, USN_DOHODY_RASHODY, OSN, ESHN}` per tenant; для NPD — flow без ККТ через **Мой налог API**.

### 2.8 SLO отправки чека

- 54-ФЗ: чек должен быть **отправлен в момент расчёта**.
- Электронная форма — на email/SMS до момента расчёта или сразу после (практика ФНС: до 5 минут).
- Offline-режим: ККТ автономно до **30 дней**.

## 3. Туристический налог 2% Сочи 2026 (425-ФЗ + НК глава 33.1)

### 3.1 Канон

```
tax = max(rooms_revenue_excluding_vat × 2%, 100 ₽/сутки × nights × rooms)
```

- **База**: стоимость услуги по предоставлению мест временного проживания, **без НДС**, **без самого тур.налога**.
- **Ставка 2026 в Сочи**: **2%** (растёт до 5% к 2029).
- **Минимум**: **100 ₽/сутки за номер**.
- **Не входит в базу**: завтрак, парковка, доп. услуги.

### 3.2 Освобождения (ст. 418 НК)

- Герои СССР/РФ.
- Участники ВОВ + блокадники.
- Инвалиды 1, 2 групп + дети-инвалиды.
- Несовершеннолетние.
- Военнослужащие, мобилизованные, ветераны боевых действий.
- Прибывшие на лечение в санаторий по бюджетной путёвке (**льгота-145**, наш `tourism-tax` это уже учитывает).
- Местные жители (Сочи: **жители Краснодарского края**).

### 3.3 Сроки уплаты + декларация

- Налогоплательщик — **отель**.
- **Налоговый период — квартал**.
- **Срок уплаты — до 28 числа** месяца после квартала.
- **Декларация — КНД 1153008** (приказ ФНС от 19.12.2025), срок **до 25 числа**.
- Подача — электронно через ФНС-кабинет / 1С / Контур.

### 3.4 Перерасчёт при cancel/no-show

- При отмене ДО заселения — тур.налог НЕ начисляется.
- При no-show с удержанием 1 ночь — тур.налог НАЧИСЛЯЕТСЯ за 1 ночь (есть фактическое право проживания).
- При partial-refund после check-in — пропорциональный refund.
- В коде: уже реализовано в M7.A.4.

### 3.5 Multi-property

- Каждый объект = отдельная строка декларации.
- Один ИНН на ИП/юрлицо, КПП может различаться для филиалов.

### 3.6 Sочи specifics

- Ставка 2% в 2026 (выше многих регионов).
- Освобождение для жителей Краснодарского края — нужна верификация по адресу регистрации.
- Гостевые дома — **облагаются** (после ФЗ-93 от 24.04.2025).

## 4. 109-ФЗ + Миграционный учёт + ЕПГУ

### 4.1 Сроки

- Постановка ИГ: **1 рабочий день** с момента заселения.
- Снятие: до **12:00 рабочего дня после выезда**.
- Регистрация граждан РФ из других регионов: **24 часа** (приказ МВД 984).

### 4.2 Постановление №1668

- Электронное взаимодействие через ЕПГУ закреплено как основной канал.
- **Соглашение с ОВМ МВД** обязательно до начала использования API. Срок: **2-6 недель**.
- Проверка через РКЛ обязательна с 5.02.2025 — на этапе **check-in**, не booking.

### 4.3 Госпошлина 500 ₽ (с 1.09.2025)

- За постановку ИГ на учёт (платит **отель**, не гость).
- Включается в себестоимость, не reimbursable.
- При ошибочной подаче — не возвращается.

### 4.4 Эксперимент «Амина»

- Альтернативный канал для отелей-участников (определённые регионы).
- **Сочи входит в перечень** (постановление 822 от 2024).
- Push-уведомление в Амина → биометрия → автоматическая постановка.
- Для нашего SaaS приоритет: API ЕПГУ через «Скала-ЕПГУ».

### 4.5 Штрафы (КоАП 18.9)

| Нарушение | Юрлицо |
|---|---|
| Несвоевременная постановка ИГ | **400–500к** |
| Не подал данные | **400–500к** |
| Невнесение в гост. реестр | **300–500к** |
| Размещение без РКЛ-проверки | **до 500к** |

## 5. ПП-1912 от 27.11.2025 (с 01.03.2026)

### Ключевые требования

- **Все бронирования возвратные** — full-refund cancellation policy default.
- **1-night cap** для удержания при no-show.
- **Hold reservation до 12:00 следующего дня** — отель обязан удерживать номер.
- **Mandatory disclosure** при booking widget:
  - Полная стоимость с НДС и тур.налогом.
  - Категория объекта (звёзды).
  - Условия отмены.
  - Адрес и контакты.
  - Реквизиты исполнителя (ИНН, ОГРН).
  - Перечень включённых услуг.
- **B2B**: ПП-1912 регулирует B2C. B2B-договоры — общегражданское регулирование.
- **ИП-гость**: для личных нужд — B2C; для бизнеса — B2B.

### Штрафы

ЗоЗПП ст. 14.8 + специальные — до 500к ₽ юрлицу.

### Реализация

- Booking widget: блок «Условия» с обязательным disclosure (axe-core: aria-required text).
- Cancellation policy enum: `{ FREE_CANCELLATION, NON_REFUNDABLE_DISCOUNTED, FREE_UNTIL_DATE }` — последний default.
- No-show fee max: hard cap = 1 night.

## 6. ПП-1951 от 27.12.2024 — Классификация и Реестр КСР

### 6.1 Реестр КСР

- **Обязательное вхождение в Единый реестр КСР до 1.09.2025**.
- Без записи — **запрет на оказание услуг** + штраф **300–450к ₽** юрлицу (КоАП 14.39).
- Реестр ведёт Минэкономразвития, портал — `classification.tourism.gov.ru`.
- Самооценка по чек-листу или классификация через аккредитованную организацию.

### 6.2 Категории

- 1*–5* + «без звёзд».
- Каждая категория — конкретный список обязательных amenities.
- 5* в Сочи — повышенные требования.

### 6.3 Реализация

- Tenant onboarding: поле `ksr_registry_id` + `category` + `category_valid_until`.
- Pre-publish booking widget gate: `if not ksr_registry_id → block`.

## 7. ЗоЗПП — гостиничные услуги

### 7.1 Refund 10 дней (ст. 22)

- Возврат потребителю — **в течение 10 календарных дней**.
- **Пеня — 1% за каждый день**.
- **Кап с 1.02.2026** (ФЗ-256): пеня **не более стоимости услуги**.

### 7.2 Замены ПП-1085

- **ПП-1085 (2015) — отменено**.
- Заменено: **ПП-1853 от 18.11.2020** + **ПП-1912 (2025)** (с 1.03.2026).

### 7.3 Право на отказ (ст. 32)

- Потребитель вправе отказаться с компенсацией исполнителю **фактических расходов**.
- В hospitality «фактические расходы» = себестоимость, на практике no-show fee = 1 night cap.

### 7.4 Жалобы

- Срок ответа на письменную претензию — **10 дней**.
- Книга отзывов с 1.01.2021 **необязательна**, но рекомендуется.

## 8. Прочие compliance

### 8.1 ЕГАИС (если бар)

- Учёт алкоголя через ЕГАИС — для всех розничных продавцов.
- Лицензия Росалкогольрегулирования (Минфин с 2024).
- Phase 3+ для нашего SaaS.

### 8.2 Меркурий (ВетИС)

- Закупка мяса/рыбы/молочки для ресторана — **ВЭД-сертификаты в Меркурии**.
- Не требуется для готовых блюд от кейтеринга.
- Phase 3+.

### 8.3 Лицензия на алкоголь

- Розничная продажа: лицензия Минфина Краснодарского края.
- 65к ₽/год за объект.
- Запрет 23:00–08:00 в Краснодарском крае.
- Зона запрета 100м от школ/больниц.

### 8.4 СанПиН + Пожарка

- **СанПиН 2.1.3678-20**.
- ФЗ-123 пожарной безопасности.
- Декларация на каждый объект.

### 8.5 Налоговая отчётность tenants

- **УСН Доходы 6%** — наиболее популярный.
- **УСН Доходы-Расходы 15%** — если высокая доля затрат.
- **ОСН** — для крупных, с НДС.
- **ПСН** — патент гостиничный.
- **НПД** — гостевые дома, лимит 2.4 млн ₽/год.
- **АУСН** (эксперимент) — расширен на все регионы с 2025-01.

### 8.6 Договор с гостем

- ПП-1853: договор может быть в письменной или электронной форме.
- Достаточно регистрационной карточки + согласия с офертой (checkbox + IP-fix + timestamp).
- Электронный договор = публичная оферта + акцепт через бронирование.

### 8.7 Реклама (38-ФЗ)

- Booking widget = реклама (применяется ФЗ-38).
- С 1.09.2022 — обязательная маркировка рекламы (ОРД-токен).
- Наш widget ≠ реклама **если** на собственном сайте отеля.
- Если widget на нашем aggregator-сайте — реклама → ОРД-токен.

### 8.8 Cookies / трекеры

- 152-ФЗ: cookies-идентификаторы — это ПД.
- Уведомление-баннер: «Сайт использует cookies» + согласие до загрузки non-essential cookies.
- **Я.Метрика** — российский провайдер, ОК.
- **GA4** — ПД утечка за рубеж, **нельзя** без явного согласия + риск.
- В коде: cookie-consent banner перед загрузкой Я.Метрики.

### 8.9 Push notifications

- Opt-in обязателен (152-ФЗ + 38-ФЗ).
- Спам = штраф ст. 14.3 КоАП до 500к ₽.

### 8.10 Антикоррупция (273-ФЗ)

- Для SaaS-вендора > 50 сотрудников — обязательно положение об антикоррупции.
- Мы пока < 50 — не критично.

### 8.11 Проверки Роспотребнадзора

- Книга жалоб/отзывов.
- Документы о согласиях ПД.
- Договоры с гостями (электронные с подписью акцепта).
- КСР-сертификат.
- Декларации по тур.налогу.
- Протоколы СанПиН.

## 14. Ключевые «не пропустить» рекомендации

1. **ОТДЕЛЬНОЕ согласие 152-ФЗ** — переделать любые UI с галочкой «Согласен с офертой и обработкой ПД» на **два отдельных шага**. Блокирующий compliance-баг с 1.09.2025.
2. **24-часовой breach reporting** — добавить в Monium runbook + дежурного юриста.
3. **vat_code 5 для room с 1.01.2026** — verify в payment-домене.
4. **Тур.налог в Сочи 2% (не 1%)** — обновить ставку.
5. **ПП-1912 disclosure block** — обязательно перед confirm-booking, axe-проверка.
6. **РКН-уведомление до начала обработки ПД** — должно быть выполнено уже сейчас (если SaaS обрабатывает реальные ПД в стейджинге — формально нарушение).
7. **DPA с YC** — проверить что в текущем договоре пункт «обработка ПД по поручению».
8. **Cookie consent banner** — отсутствие = риск 30-50к за нарушение РКН.

Эти восемь — **must-have** до любого public-launch демо.
