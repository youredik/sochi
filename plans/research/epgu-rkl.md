## 1. Юридический канон 2026

### 1.1 Постановление №1668 — что РЕАЛЬНО ввели

**Постановление Правительства РФ от 27.10.2025 № 1668** (ред. 27.11.2025). **НЕ вводит "обязательность ЕПГУ для всех" с 1 января 2026** — это распространённый миф из вторичных публикаций. Реально:

- **С 1 января 2026** граждане РФ при заселении могут предъявлять, помимо паспорта: загранпаспорт РФ, паспорт СССР, временное удостоверение, свидетельство о рождении (до 14 лет), **водительское удостоверение** (МВД проверяет онлайн через ИС ГИБДД).
- **С 1 марта 2026** — удостоверение личности военнослужащего.
- **Перерегистрационный срок** для ИГ — в течение 1 рабочего дня после прибытия (без изменений по сравнению с 109-ФЗ).
- Срок постановки граждан РФ из других регионов — 24 часа; де-регистрация — до 12:00 рабочего дня после выезда. Хранение документации — 1 год.

### 1.2 Штрафы (ст. 18.9 КоАП РФ для юрлиц 2026)

- За неуведомление о прибытии иностранца: **400 000–500 000 ₽** за каждого гостя (Москва/СПб/курортные регионы — до **750 000 ₽**).
- За заселение иностранца, состоящего в РКЛ: до **500 000 ₽** + риск приостановления деятельности до 90 суток.
- За неуведомление по гражданам РФ (ст. 19.15.2): **250 000–750 000 ₽** для юрлиц.

В вторичных источниках есть проект ужесточения (80–500к ₽ за каждого ИГ при просрочке) — на 27.04.2026 это **проект**, не закон.

### 1.3 Каналы передачи данных — три легальных пути

1. **Прямая интеграция к API ЕПГУ** через TLS ГОСТ — путь TravelLine, Контур, Bnovo. Своё соглашение с МВД + КЭП.
2. **Скала-ЕПГУ** — посредник, ~720 ₽/мес базовый тариф, свой web-кабинет.
3. **СВОКС** (Защищённая сеть) — корпоративный канал.

С 2027 г. (по сообщениям, неподтверждено приказом) приоритетным становится СВОКС, текущие подключения через ГОСТ TLS работают до 31.12.2026.

## 2. API ЕПГУ — техническая спецификация v1.3

**Спецификация:** "API ЕПГУ. Сервисы миграционного и регистрационного учётов для гостиниц", версия **1.3 (19.12.2022)**, gu-st.ru. Более новой версии на 27.04.2026 не опубликовано.

| Production | `https://www.gosuslugi.ru/412331/1/form` | `.../412331/2/form` |

### 2.2 REST-методы (через SMEV4-обёртку)

```
POST /gu-smev-api/api/gusmev/order            # резерв orderId
POST /gu-smev-api/api/gusmev/push/chunked     # подача архива (multipart)
POST /gu-smev-api/api/gusmev/order/{orderId}  # запрос статуса
```

### 2.3 Авторизация

OAuth 2.0 client_credentials через ЕСИА. `Authorization: Bearer {token}` + cookie `acc_t={token}`. Token TTL — в практике партнёров ~1800 секунд для access_token.

### 2.4 Ключевые eServiceCode/serviceTargetCode

- **Постановка ИГ**: `eServiceCode = 10000103652`, `serviceTargetCode = -1000444103652`
- **Снятие ИГ**: `eServiceCode = 10000103653`, `serviceTargetCode = -1000444103653`
- **Регистрация/снятие граждан РФ**: `eServiceCode = 10000103654`

## 3. Lifecycle и тайминги

### 3.1 Статусы внутри услуги «Постановка ИГ»

| Код | Название | Final? |
|---|---|---|
| 0 | Черновик | нет |
| 1 | Принято от заявителя | intermediate |
| 2 | Отправлено в ведомство | intermediate |
| 3 | **Исполнено** | **final** |
| 4 | **Отказ** | **final** |

### 3.2 Тайминги (из практики Контур + TravelLine)

- **Ack** (получение `orderId` после `push/chunked`): синхронно, 1-3 сек.
- **Переход 1→2** (СМЭВ): 1-2 минуты.
- **Переход 2→3/4** (МВД): **5–20 минут типовой случай**, P95 = 20 мин, P99 = 60 мин, в простое МВД — до утра следующего дня. Регламент SLA до **5-7 рабочих дней** для асинхронных видов сведений.
- **5-10% случаев** МВД не передаёт статус снятия с учёта — нужна ручная сверка через ЛК Госуслуг.

### 3.3 Polling vs webhook

**Только polling.** WebHook-нотификаций со стороны ЕПГУ для гостиничных сервисов нет. Канонический паттерн:

1. После `push` фиксируем `orderId`, статус 17.
2. Polling `getOrdersStatus` или `order/{orderId}` — рекомендация: 1 мин первые 10 мин, 5 мин до часа, экспоненциально дальше.
3. `getUpdatedAfter` для batch-обновления статусов всех активных orderId.

## 4. ЭЦП и сертификаты

- **Алгоритм:** ГОСТ Р 34.10-2012 (256 бит) + ГОСТ Р 34.11-2012 для хеширования. TLS — ГОСТ.
- **Тип:** УКЭП (усиленная квалифицированная). Каждое юрлицо — свой; каждая гостиница (если разные адреса) — отдельный.
- **Источник:** УЦ ФНС (бесплатно для руководителей ЮЛ/ИП с 01.01.2022, **срок 15 месяцев**) или аккредитованные доверенные лица. Носитель Rutoken/JaCarta ~1500-3500 ₽.
- **Криптопровайдер:** КриптоПро CSP 5.0+ (de-facto обязательный). Лицензия ~2700 ₽/год.
- **Подписание:** PKCS#7 detached, для каждого файла в архиве своя `.sig`.

## 5. Коды ошибок (для эмуляции в моке)

**Уровни:**

1. **Транспортные (HTTP):** 400, 401, 403, 413, 429, 500, 503.
2. **СМЭВ-уровень:** SRV001 (Орган недоступен), SRV002 (Невалидная подпись), SRV003 (Невалидный XML), SRV004 (Конфликт ID), SRV010 (Неизвестный orderId).
3. **Уровень ИС МВД** (свободный текст в `<reasonRefuse>`):
   - **Неверно заполнены данные** (несоответствие маски ФЛК)
   - **Сертификат не прошёл проверку**
   - **Дубль уведомления** (повторная подача с тем же applicationNumber)
   - **Документ в реестре утраченных/недействительных**
   - **Иностранный гражданин в РКЛ** — отказ в постановке
   - **Подразделение МВД не обслуживает регион** — ошибка маршрутизации
   - **Срок пребывания за пределами разрешённого** (более 90/180 для безвиз)
   - **Истёкший паспорт**

**Для mock** разумно эмулировать **8 категорий**: `validation_format`, `signature_invalid`, `duplicate_notification`, `document_lost_or_invalid`, `rkl_match`, `region_mismatch`, `stay_period_exceeded`, `service_temporarily_unavailable`.

## 6. РКЛ (реестр контролируемых лиц)

### 6.1 Юр.канон

- **Указ Президента РФ № 1126** (декабрь 2024) — установил режим высылки и понятие «контролируемое лицо».
- **С 5 февраля 2025** РКЛ публично работает на мвд.рф/rkl и доступен через СМЭВ-рассылку.
- **С 5 февраля 2025** для всех средств размещения — **обязательная проверка** каждого иностранного гостя на въезд (ст. 24.10 ФЗ-115).
- При обнаружении match — **немедленно** сообщить в территориальный ОВМ МВД и не оказывать услуги размещения.

### 6.2 Каналы доступа в 2026

| Канал | Тип | Стоимость | Sandbox |
|---|---|---|---|
| Прямая проверка через мвд.рф/rkl (HTML) | Public, CAPTCHA | бесплатно | нет |
| СМЭВ3 «Рассылка реестра» | через ЕСНСИ | бесплатно | да (PAIP) |
| **Контур.ФМС API** | По договору | ~3000 ₽/мес | по запросу |
| СКАЛА-ЕПГУ модуль | По договору | ~720 ₽/мес базовый | нет |
| РосМигрант / RKL Scan | API + batch | договорная | да |

**Прямого МВД API для отелей нет** (только web с CAPTCHA). Все программные пути — через СМЭВ или посредников.

### 6.3 СМЭВ3 формат рассылки

**Файл:** CSV в ZIP, UTF-8. Поля каждой записи:
- Уникальный идентификатор записи реестра
- Идентификатор цифрового профиля (опционально)
- День, месяц, год рождения
- Серия/номер документа
- Дата выдачи документа

**Обновление:** через S3 или FTP. Каждая ревизия — полный снимок, без истории. По факту — 1-3 раза в сутки.

**Отсутствует** в публичной рассылке: ФИО, гражданство, причина включения. Они есть только в защищённой версии для МВД и коммерческих интеграторов.

## 7. Подводные камни (от конкурентов и интеграторов)

1. **Соглашение об информационном взаимодействии** (приказ МВД №435 от 09.07.2018) обязательно и заключается с **территориальным ОВМ МВД региона** по факт-адресу гостиницы. Без него API-канал не работает. Срок заключения — **2-6 недель**.
2. **`supplierGid` и `userId`** выдаются МВД при заключении соглашения. Без них — ошибка маршрутизации.
3. **Гостиница на нескольких адресах** = несколько соглашений + несколько `supplierGid` + несколько КЭПов.
4. **МВД не возвращает статус снятия в 5-10% случаев** — нужна ежедневная сверка через ЛК.
5. **Эксперимент «Амина»** (с 01.09.2025): если ИГ участвует в эксперименте, уведомление о прибытии гостиницей **не подаётся**. Mock должен это распознавать.
6. **Госпошлина 500 ₽ за постановку ИГ** введена с 01.09.2025 — новая статья расходов принимающей стороны.
7. **Несоответствие наименования гостиницы** (символ-в-символ) → отказ. Кавычки `"` vs `«»` ломают подачу.

## 8. Решения для нашего mock-адаптера

### 8.1 Минимальный набор behaviour-faithful поведений

1. **Двухфазный submit:** `reserveOrderId` (1-3 сек) → `pushArchive` (2-5 сек multipart) → возврат `orderId`.
2. **Async lifecycle:** статус 17 (моментально) → 21 (30-90 сек) → 1 (1-2 мин) → 2 (2-5 мин) → 3 или 4 (P95=20 мин, P99=60 мин, P999=несколько часов).
3. **Polling-only:** никаких webhook, только GET статусов; интервалы клиента 1 мин/5 мин/экспоненциально.
4. **8 категорий ошибок** для статуса 4 со свободным текстом в `reasonRefuse`.
5. **РКЛ-проверка** до постановки: clean/match/inconclusive с локальным CSV-snapshot, имитировать обновление 2-3 раза в сутки.
6. **5-10% "lost confirmation"** для снятия с учёта — намеренно не присылать финальный статус, требовать ручную сверку.
7. **Дубль-детект** по `applicationNumber` + (`supplierGid`, иностранец+документ+дата прибытия) — мгновенный отказ "duplicate_notification".
8. **Регламентные простои** — 1 раз в 2 недели на 30 минут возвращать HTTP 503 с `Retry-After: 1800`.
9. **Подпись:** валидация `.sig` PKCS#7 detached на каждый файл; mock проверяет только присутствие, валидацию ГОСТ-подписи пропускает с warning.
10. **Тестовые данные:** `supplierGid=5200000008002021`, `regionCode=f9ca8229-...-948eb08f2700` (как в спецификации) — sandbox-маркер.

### 8.2 РКЛ mock

- 99% случаев — `clean`.
- 0.5% — `match` (требует ручной проверки + блокировка постановки).
- 0.5% — `inconclusive`.
- Задержка 50-300 мс.
- Хранит in-memory snapshot, симулирует ежедневное обновление (timer-based).

## 9. Открытые вопросы (нужно прояснить до реальной интеграции)

1. Точные rate-limits для `getOrdersStatus` и `push/chunked` (Минцифры публикует только общие принципы).
2. Полный canonical-список кодов ошибок ИС МВД (нет публичного реестра).
3. Точная XSD-схема `attach.xml` (публикуется только в ППО «Территория» закрытом партнёрском интерфейсе).
4. Точный момент перехода на СВОКС как обязательный.
5. Webhook-нотификации в дорожной карте СМЭВ4 — даты в публичной документации нет.
6. Контур.ФМС реальная схема API (под NDA).
7. Эксперимент «Амина» — точные требования к гостинице.
8. РКЛ-CSV точный column order и разделитель.
9. SLA отказа МВД при дубле уведомления.
